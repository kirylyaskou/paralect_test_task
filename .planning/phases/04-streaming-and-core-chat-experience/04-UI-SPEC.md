# Phase 4 — UI Design Contract

> Visual and interaction contract for the core chat experience: message list, streaming responses, markdown rendering, welcome screen, chat input, error handling. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

| Property | Value |
|----------|-------|
| Tool | shadcn v4 |
| Preset | base-nova / neutral base color |
| Component library | Base UI (`@base-ui/react`) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, code block copy button inset |
| sm | 8px | Gap between avatar and message bubble, gap between messages of same role |
| md | 16px | Gap between message groups (different roles), welcome card inner padding, input padding, prompt card grid gap, user message bubble padding |
| lg | 24px | Message list horizontal padding (desktop), welcome screen section spacing |
| xl | 32px | Welcome screen vertical padding, chat input area padding from edges |
| 2xl | 48px | Welcome heading to prompt grid gap |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch target for send/stop button is 44px (accessibility minimum for touch targets). Scroll-to-bottom FAB is 40px diameter.

## Typography

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 14px (text-sm) | 400 (normal) | 1.6 | Message text, markdown prose, suggested prompt descriptions |
| Label | 14px (text-sm) | 600 (semibold) | 1.4 | Message role labels (if shown), code block language tag, suggested prompt titles |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Welcome screen heading, empty state heading |
| Caption | 12px (text-xs) | 400 (normal) | 1.5 | Message timestamps, "AI is thinking..." indicator, error helper text, token streaming indicator |

**Weight constraint:** This phase uses exactly 2 weights -- 400 (normal) for body content, captions, and markdown prose, 600 (semibold) for labels and headings. No other weights are permitted.

**Markdown-specific typography:**

| Element | Style |
|---------|-------|
| Inline code | `font-mono`, 12px, `--muted` background with 4px horizontal padding, `rounded-sm` |
| Code block | `font-mono`, 12px, `--muted` background, 16px padding, `rounded-md`, full width |
| Code block header | 12px `font-mono`, `--muted-foreground` text for language label, copy button right-aligned |
| Bold | weight 600 (semibold) -- not 700 |
| Headings in markdown | h1=20px, h2=16px semibold, h3=14px semibold -- all using `--foreground` |
| Lists | 14px body, 8px gap between items, 16px left indent |
| Blockquote | 14px body, `--muted-foreground` text, 2px left border in `--border` color, 16px left padding |
| Links in markdown | `--primary` color, underline on hover |
| Paragraphs | 12px bottom margin between paragraphs |

## Color

All colors use the existing shadcn CSS custom properties defined in `app/globals.css`. Do NOT introduce new color variables.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` | Chat area background, message list background |
| Secondary (30%) | `--muted` / `--card` | User message bubble background, code block background, welcome prompt cards, input area background |
| Accent (10%) | `--primary` / `--primary-foreground` | Send button, scroll-to-bottom FAB, suggested prompt hover state, user message avatar background |
| Destructive | `--destructive` | Error toast icon, failed message indicator, retry button text |
| Muted | `--muted-foreground` | Timestamps, placeholder text in input, "AI is thinking" text, assistant avatar background |
| Border | `--border` | Input border, welcome prompt card border, code block border |

Accent reserved for: send message button, scroll-to-bottom FAB background, suggested prompt card hover border, user message avatar background. No other elements use `--primary` in this phase.

**Message bubble colors:**

| Role | Background | Text |
|------|------------|------|
| User | `--muted` | `--foreground` |
| Assistant | `transparent` (no bubble) | `--foreground` |

This matches ChatGPT convention: user messages have a subtle bubble, assistant messages render directly on the page background for better readability of long-form markdown content.

## Focal Point

The message list is the primary visual area, occupying the full remaining viewport height between the content header and the chat input. The chat input bar at the bottom is the secondary focal point -- always visible, anchored to the bottom of the viewport. The welcome screen (when no messages exist) centers itself vertically in the message area with the heading as the dominant text element and the 2x2 prompt grid as the interactive anchor.


**Layout:**
- Message list: `flex-1 overflow-y-auto`, horizontal padding 24px on desktop / 16px on mobile
- Messages max-width: 768px, centered within the message area (mx-auto)
- Input area: sticky bottom, full width, 16px padding, `--background` with top border `--border`
- Input max-width: 768px, centered (matches message width)

**Scroll behavior:**
- Auto-scroll to bottom when new tokens arrive during streaming
- If user scrolls up more than 100px from bottom, auto-scroll pauses
- Scroll-to-bottom FAB appears when not at bottom (40px circle, `--primary` background, Lucide `ArrowDown` icon, bottom-right of message area, 16px from edge)
- Clicking FAB smooth-scrolls to bottom and resumes auto-scroll

## Interaction States

### Chat Input (Textarea)

| State | Visual Treatment |
|-------|-----------------|
| Default | `--border` border, `--background` fill, placeholder "Type a message..." in `--muted-foreground` |
| Focus | `--ring` border via focus-visible ring (shadcn default) |
| Typing | Auto-resize from 1 row to max 6 rows, then scroll internally |
| Disabled (during streaming) | 50% opacity, `pointer-events-none` |

### Send Button

| State | Visual Treatment |
|-------|-----------------|
| Default (empty input) | `--primary` background, 50% opacity, disabled -- cannot send empty message |
| Default (has input) | `--primary` background, `--primary-foreground` icon (Lucide `Send`), fully opaque |
| Hover | `--primary/80` background (shadcn default hover) |
| During streaming | Transforms to Stop button: `--destructive` text, ghost variant, Lucide `Square` icon |

### Send/Stop Button Size
- 44px touch target (min), icon size 20px
- Positioned at bottom-right of textarea

### Message Bubbles

| State | Visual Treatment |
|-------|-----------------|
| User message | `--muted` background, `rounded-2xl`, 16px vertical padding, 16px horizontal padding, max-width 85% of container, right-aligned |
| Assistant message | No background, left-aligned, full width up to 768px max |
| Streaming (assistant) | Same as assistant but with blinking cursor (1px wide, `--foreground`, `animation: blink 1s step-end infinite`) appended after last token |
| Error (assistant) | Red-tinted: `--destructive/10` background, `--destructive` left border (2px), retry button below |

### Message Avatars

| Role | Visual Treatment |
|------|-----------------|
| User | 32px circle, `--primary` background, `--primary-foreground` text, Lucide `User` icon (16px) |

### Suggested Prompt Cards

| State | Visual Treatment |
|-------|-----------------|
| Default | `--card` background, `--border` border, `rounded-lg`, subtle shadow-sm |
| Hover | `--accent` background, `--primary` border color, cursor pointer |

### Scroll-to-Bottom FAB

| State | Visual Treatment |
|-------|-----------------|
| Hidden | `opacity-0 pointer-events-none` with transition |

| State | Visual Treatment |
|-------|-----------------|

## Loading States

| Scenario | Visual Treatment |
|----------|-----------------|
| Chat history loading | 3 skeleton message rows: alternating left/right alignment, skeleton avatar (32px circle) + skeleton text lines (2-3 lines, varying widths 40-80%) |
| Waiting for first token | Streaming indicator (animated dots) in assistant message area |
| Sending message | User message appears immediately (optimistic), input clears, input disabled |
| Title generating | No visible loading state -- sidebar title updates silently when ready |

## Error Handling Contract

| Error Scenario | User Experience |
|----------------|-----------------|
| Network error on send | Sonner error toast with "Failed to send message. Check your connection and try again." User message remains in input. |
| Streaming interrupted | Partial assistant message shown with error styling (destructive border). Inline "Retry message" button below the message. |
| API returns 401 | Redirect to `/login` (session expired). |
| API returns 429 | Sonner error toast with "Too many requests. Please wait a moment and try again." |
| API returns 500 | Sonner error toast with "Something went wrong. Please try again." |
| Retry clicked | Re-sends the original user message, replaces the error message with new streaming response. |
| Title generation fails | Silent failure -- chat keeps default null title, sidebar shows "New chat" fallback. No toast. |
