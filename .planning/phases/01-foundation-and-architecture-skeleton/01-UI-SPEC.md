# Phase 1 — UI Design Contract

> Visual and interaction contract for the foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 1 scope note:** This phase scaffolds the project, initializes Shadcn/ui, configures Tailwind CSS 4 theming, and sets up providers. There is no feature UI -- no pages, no interactive components, no user-facing flows. This contract defines the **design tokens and conventions** that all subsequent phases will consume. The only visible output is a minimal root layout that renders the provider tree and confirms the build works.

---

| Property | Value |

### shadcn Initialization Notes

shadcn has not been initialized yet (`components.json` does not exist). Phase 1 execution must run:

```bash
pnpm dlx shadcn@latest init
```

Accept defaults: New York style, neutral base color, OKLCH color variables. This creates `components.json`, installs `tw-animate-css`, and scaffolds theme variables in `globals.css`.

No shadcn components need to be added in Phase 1. Components will be added on-demand in Phase 2+ (e.g., `button`, `input`, `card`, `sheet`, `dialog`, `toast`, `skeleton`).

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, input internal padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, sidebar internal spacing |
| xl | 32px | Layout gaps between major regions |
| 2xl | 48px | Major section breaks, page top/bottom padding |
| 3xl | 64px | Page-level vertical spacing, hero areas |

Exceptions: none

**Implementation:** Tailwind v4 provides these via the default spacing scale (e.g., `p-1` = 4px, `p-2` = 8px, `p-4` = 16px, `p-6` = 24px, `p-8` = 32px, `p-12` = 48px, `p-16` = 64px). No custom theme extension needed -- the default Tailwind v4 scale covers all tokens.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base font-normal leading-normal` |
| Label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold leading-snug` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |
| Display | 28px | 600 (semibold) | 1.15 | `text-[28px] font-semibold leading-tight` |

**Font stack:** Inter with system fallback. Loaded via `next/font/google` in `app/layout.tsx` for zero layout shift.

```typescript
// app/layout.tsx
import { Inter } from 'next/font/google'
const inter = Inter({ subsets: ['latin'] })
```

**Weight constraint:** Exactly 2 design-token weights are used: **400 (regular)** for body text and **600 (semibold)** for labels, headings, and display text. The `next/font/google` loader may technically load additional weight subsets for rendering optimization, but the design contract restricts all UI elements to these two weights only.

**Source:** RESEARCH.md Code Examples (root layout shows Inter). Weight/size scale is a default recommendation for chat application readability.

## Color

Colors use Shadcn/ui OKLCH variables defined in `globals.css`. The values below represent the semantic roles mapped to shadcn's CSS custom properties.

### Light Mode

| Role | CSS Variable | Approximate Hex | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | #ffffff | Page background, main content area |
| Secondary (30%) | `--card`, `--muted` | #f9fafb / #f1f5f9 | Cards, sidebar background, nav, input backgrounds |
| Accent (10%) | `--primary` | #18181b (near-black) | Primary CTA buttons, active nav indicators, focused input borders |
| Destructive | `--destructive` | #ef4444 | Delete chat button, error states only |

### Dark Mode

| Role | CSS Variable | Approximate Hex | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | #09090b | Page background, main content area |
| Secondary (30%) | `--card`, `--muted` | #171717 / #262626 | Cards, sidebar background, nav, input backgrounds |
| Accent (10%) | `--primary` | #fafafa (near-white) | Primary CTA buttons, active nav indicators, focused input borders |
| Destructive | `--destructive` | #ef4444 | Delete chat button, error states only |

**Accent reserved for (exhaustive list across all phases):**
- Primary action buttons ("Send", "New Chat", "Sign In", "Create Account")
- Active/selected chat item in sidebar
- Focused input border ring
- Link text on hover
- Toggle switch active state (theme toggle)

**NOT accent:** Navigation text, card borders, dividers, disabled states, skeleton loading bars. These use `--muted-foreground` or `--border`.

**Implementation:** Shadcn init generates these variables automatically in `globals.css` using `@theme inline`. Do not override the defaults in Phase 1 -- the neutral palette with near-black/near-white primary is appropriate for a chat application (content-focused, minimal distraction). If accent color needs adjustment in later phases, modify the `--primary` OKLCH values in `globals.css`.

**Source:** RESEARCH.md confirms OKLCH color space, Shadcn/ui Tailwind v4 theming. Default neutral palette is appropriate for a ChatGPT-like interface.

### Conventions for Future Phases (established here, consumed by Phases 2-5)

| Element | Copy | Phase |
|---------|------|-------|

| Primary CTA (chat) | "Send message" | Phase 4 |

## Phase 1 Specific: What Gets Built Visually

Phase 1 is infrastructure-only. The visual deliverables are limited to:

1. **`app/globals.css`** -- Tailwind v4 imports + Shadcn theme variables (OKLCH, light/dark mode). Generated by `shadcn init`.
2. **`app/layout.tsx`** -- Root layout with Inter font, `<html lang="en" suppressHydrationWarning>`, `Providers` wrapper. Dark mode class applied by `next-themes`.
3. **`app/providers.tsx`** -- Client component with `QueryClientProvider` + `ThemeProvider` (attribute="class", defaultTheme="system", enableSystem).
4. **`app/page.tsx`** -- Minimal placeholder page confirming the app builds and renders. No interactive elements.

**No interactive components, no forms, no navigation.** These arrive in Phase 2+.
