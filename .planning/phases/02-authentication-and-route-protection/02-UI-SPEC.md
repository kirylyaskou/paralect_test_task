## Typography

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 (leading-snug) | Form field descriptions, helper text |
| Label | 14px (text-sm) | 600 (semibold) | 1.4 | Form labels, link text |
| Heading | 24px (text-2xl) | 600 (semibold) | 1.2 (leading-tight) | Page title ("Create an account", "Welcome back") |
| Caption | 12px (text-xs) | 400 (normal) | 1.5 | Validation error messages below fields |

**Weight constraint:** This phase uses exactly 2 weights -- 400 (normal) for body content and captions, 600 (semibold) for labels and headings. No other weights are permitted.

## Color

All colors use the existing shadcn CSS custom properties defined in `app/globals.css`. Do NOT introduce new color variables.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` | Page background behind auth card |
| Secondary (30%) | `--card` / `--card-foreground` | Auth card surface |
| Accent (10%) | `--primary` / `--primary-foreground` | Submit buttons only ("Create account", "Sign in") |
| Destructive | `--destructive` | Inline validation error text, failed-submission alert icon |
| Muted | `--muted-foreground` | Placeholder text, secondary descriptive copy, navigation links |
| Border | `--border` | Card border, input field borders |

Accent reserved for: primary CTA button on login page, primary CTA button on signup page. No other elements use `--primary` in this phase.

## Focal Point

The page heading (e.g., "Welcome back" / "Create an account") is the primary visual anchor, drawing the eye via the largest type size (24px semibold) at the top of the card. The submit button is the secondary focal point, distinguished by the accent color (`--primary`) against the neutral card surface.

## Page Layouts

### Login Page (`/login`)

```
+--------------------------------------------------+
|                                                  |
|              [background surface]                |
|                                                  |
|           +----------------------------+         |
|           |  [App logo / name]         |         |
|           |  Heading: "Welcome back"   |         |
|           |  Subtext: "Sign in..."     |         |
|           |                            |         |
|           |  [Email label]             |         |
|           |  [Email input         ]    |         |
|           |                            |         |
|           |  [Password label]          |         |
|           |  [Password input      ]    |         |
|           |                            |         |
|           |  [====  Sign in  ====]     |         |
|           |                            |         |
|           |  Don't have an account?    |         |
|           |  Create account (link)     |         |
|           +----------------------------+         |
|                                                  |
+--------------------------------------------------+
```

**Layout:** Vertically and horizontally centered card on full-viewport background. Card max-width: 400px. Card uses `rounded-lg` border radius from shadcn defaults.

**Behavior:**
- Form validates on submit (not on blur for first interaction)
- After first failed submit, subsequent changes re-validate on change
- Submit button shows loading spinner (Lucide `Loader2` with `animate-spin`) and disables during submission
- Server errors (wrong password, user not found) display in an `Alert` component above the form fields
- Inline validation errors appear as `Caption` text (12px, destructive color) below the offending field
- Successful login redirects to `/` (main chat interface)

### Signup Page (`/signup`)

```
+--------------------------------------------------+
|                                                  |
|              [background surface]                |
|                                                  |
|           +----------------------------+         |
|           |  [App logo / name]         |         |
|           |  Heading: "Create account" |         |
|           |  Subtext: "Get started..." |         |
|           |                            |         |
|           |  [Email label]             |         |
|           |  [Email input         ]    |         |
|           |                            |         |
|           |  [Password label]          |         |
|           |  [Password input      ]    |         |
|           |                            |         |
|           |  [== Create account ==]    |         |
|           |                            |         |
|           |  Already have an account?  |         |
|           |  Sign in (link)            |         |
|           +----------------------------+         |
|                                                  |
+--------------------------------------------------+
```

**Layout:** Identical structure to login page. Same max-width, same centering.

**Behavior:**
- Same validation pattern as login (validate on submit, then on change)
- Password field shows minimum requirement hint: "Must be at least 6 characters" as helper text below input (muted-foreground, text-xs) -- visible before any interaction
- Server errors (email already registered) display in Alert component
- Successful signup: allow immediate access (no email verification required -- simplifies demo flow), redirect to `/`

## Interaction States

### Input Fields

| State | Visual Treatment |
|-------|-----------------|
| Default | `--border` border, `--background` fill |
| Focus | `--ring` border via focus-visible ring (shadcn default) |
| Error | `--destructive` border, error message below in destructive color |
| Disabled | 50% opacity (shadcn default `disabled:opacity-50`) |

### Submit Button

| State | Visual Treatment |
|-------|-----------------|
| Default | `--primary` background, `--primary-foreground` text |
| Hover | `--primary/80` background (shadcn default) |
| Loading | Lucide `Loader2` icon with `animate-spin` replaces text, button disabled |
| Disabled | 50% opacity, `pointer-events-none` |

### Navigation Links ("Create account" / "Sign in")

| State | Visual Treatment |
|-------|-----------------|
| Default | `--muted-foreground` text, no underline |
| Hover | `--foreground` text, underline |

Style: Use a plain `<a>` (via Next.js `Link`) with `text-sm font-semibold text-muted-foreground hover:text-foreground hover:underline` classes. Do NOT use a Button with `variant="link"` for navigation between auth pages.

## Redirect & Session Flows

| Scenario | Behavior |
|----------|----------|
| Unauthenticated user visits protected route | Middleware redirects to `/login` (HTTP 307) -- no flash of protected content |
| Authenticated user visits `/login` or `/signup` | Middleware redirects to `/` (HTTP 307) |
| Successful login | Client-side redirect to `/` via `router.push('/')` after API response |
| Successful signup | Client-side redirect to `/` via `router.push('/')` after API response |
| Logout | Client-side redirect to `/login` via `router.push('/login')` after API response |
| Session expired (API returns 401) | Redirect to `/login` -- handled per-request, not globally in this phase |

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Form labels | Every input has a visible `<Label>` element with `htmlFor` matching input `id` |
| Error association | Error messages use `aria-describedby` linked to the input |
| Invalid state | Inputs with errors use `aria-invalid="true"` |
| Focus management | After server error, focus moves to the Alert component |
| Keyboard navigation | Tab order: email -> password -> submit button -> navigation link |
| Submit via Enter | Form submits on Enter key in any field (native form behavior) |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 ratio) -- shadcn neutral palette satisfies this |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|

Auth pages do not have a sidebar, header, or navigation bar. The card is the only UI element on the page.
