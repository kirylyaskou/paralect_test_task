# Phase 5 -- UI Design Contract

> Visual and interaction contract for file attachments (image + document), anonymous trial access modal, and multi-tab sync. Deployment and documentation have no frontend UI components. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |
|----------|-------|
| Tool | shadcn v4 |
| Preset | base-nova / neutral base color |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between attachment thumbnail and remove button, file chip icon-to-text gap |
| sm | 8px | Gap between multiple attachment thumbnails, gap between file chip elements |
| md | 16px | Attachment preview bar padding, dialog body padding, file chip padding |
| lg | 24px | Dialog section spacing, attachment preview bar margin-bottom |
| xl | 32px | Anonymous limit dialog vertical padding |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Attachment thumbnail is 64px square (not a spacing token, it is a content dimension). Touch target for attach button and thumbnail remove button is 44px minimum.

| Role | Token | Usage |

## Focal Point

The primary focal point remains the message list and chat input (unchanged from Phase 4). The new attachment preview bar sits between the message list and the input, appearing only when files are staged. The anonymous limit dialog is a blocking modal -- it becomes the sole focal point when triggered, dimming the chat behind it to force the user toward registration.

```
+--------------------------------------------------+

| +----------------------------------------------+ |
| +----------------------------------------------+ |

+--------------------------------------------------+
```

**Layout:**
- Attachment preview bar: `flex flex-wrap gap-8px`, `--muted` background, `--border` top border, 8px vertical padding, 16px horizontal padding
- Preview bar appears only when 1+ files are staged; collapses to zero height otherwise
- Preview bar max-width matches input max-width (768px, centered)
- Attach button (paperclip): positioned left of send button inside the textarea container, 44px touch target

### Message Item -- With Image Attachments

```
+--------------------------------------------------+
|                    [User avatar]  User text msg   |
|                                   in bubble       |
|                    [img1] [img2]                  |  <-- images below text
+--------------------------------------------------+
```

**Layout:**
- Images render below the user's text content, inside the user message bubble
- Image grid: `flex flex-wrap gap-8px`
- Each image: `rounded-md`, max-width 256px, max-height 256px, `object-cover`, clickable to open full-size in new tab
- Single image: renders at natural aspect ratio up to 256px max dimension
- Multiple images: thumbnail grid, each 128px square with `object-cover`

### Message Item -- With Document Attachment

**Layout:**

## File Upload Flow

| Step | What Happens |
|------|-------------|
| 1. User attaches file | Client validates type (`image/*`, `.pdf`, `.docx`) and size (< 10MB). If invalid, show toast and reject. |
| 2. File staged | Thumbnail (image) or file chip (document) appears in attachment preview bar above input. File held in client memory as `File` object. |
| 3. User sends message | Text + file(s) sent to upload API route. Files uploaded to Supabase Storage. |
| 4. Server processes | Images: store URL in `messages.image_urls[]`. Documents: extract text via pdf-parse/mammoth, store in `documents` table, URL in Supabase Storage. |
| 5. LLM receives | Images: sent as image_url parts in message. Documents: extracted text prepended to system prompt. |
| 6. Message renders | User message shows text + image thumbnails + document chips. Assistant response reflects awareness of attachments. |

---

## Anonymous Access Flow

| Step | What Happens |
|------|-------------|
| 1. Page load (unauthenticated) | FingerprintJS generates browser fingerprint. Check `anonymous_usage` table for question count. |
| 2. User sends message (count < 3) | Message processes normally. Server increments `question_count` for fingerprint. |
| 3. User sends message (count >= 3) | API returns 403 with `{ error: "limit_reached" }`. Client shows AnonymousLimitDialog. Input is disabled behind the dialog. |
| 4. User clicks "Sign Up" | Navigate to `/signup`. |
| 5. User clicks "Log In" | Navigate to `/login`. |

## Multi-Tab Sync Contract

| Event | Behavior |
|-------|----------|
| Chat created in Tab A | Tab B receives Supabase Realtime INSERT event on `chats` table. TanStack Query `['chats']` invalidated. Chat list refetches and shows new chat. |
| Chat deleted in Tab A | Tab B receives DELETE event. Query invalidated. Chat disappears from list. If Tab B was viewing the deleted chat, redirect to `/`. |
| Connection lost | Supabase Realtime auto-reconnects. No user-visible indicator needed. |
| Tab gains focus | No manual refetch -- Realtime subscription handles it. TanStack Query `refetchOnWindowFocus` remains as configured. |

|----------|-----------------|

## Error Handling Contract

| Error Scenario | User Experience |
|----------------|-----------------|
| File > 10MB selected | Sonner error toast: "File is too large. Maximum size is 10 MB." File not added to preview. |
| Invalid file type | Sonner error toast: "Unsupported file type. Please upload images, PDFs, or DOCX files." File not added to preview. |
| Upload fails (network) | Sonner error toast: "Failed to upload file. Please try again." Message not sent. Attachments remain in preview bar. |
| Document extraction fails | Sonner error toast: "Could not extract text from document. The file may be corrupted." Document still saved but without extracted text context. |
| Anonymous limit reached | AnonymousLimitDialog modal appears. Input disabled. Must navigate to signup/login. |
| Fingerprint generation fails | Fall back to random UUID per session. User still gets 3 questions but counter resets on new session. |
| Supabase Realtime disconnects | Silent auto-reconnect. No user notification. Chat list may be briefly stale. |

| Breakpoint | Behavior |

| Tablet (640-1023px) | Attachment preview bar: max-width 768px centered. Thumbnails 64px. Anonymous dialog: 400px max-width centered. Buttons side by side. |

## Accessibility Requirements

| Requirement | Implementation |

| Clipboard paste | No additional a11y needed -- standard browser paste behavior |
| Realtime updates | `aria-live="polite"` on sidebar chat list (already established in Phase 3) |
