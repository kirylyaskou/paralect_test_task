2. Create `lib/supabase/public-client.ts` -- a client-side Supabase client using the public anon key (NOT service_role). This is used ONLY for Storage `uploadToSignedUrl` and Realtime subscriptions (per architecture rules):
   ```typescript
   'use client'
   import { createClient } from '@supabase/supabase-js'

   export const supabasePublic = createClient(
     process.env.NEXT_PUBLIC_SUPABASE_URL!,
     process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
   )
   ```

3. Create `app/api/upload/route.ts` -- POST endpoint that receives file metadata and returns a Supabase Storage signed upload URL. This avoids the Vercel 4.5MB body limit (per D-04, research Pitfall 1):
   - Import `requireAuth` from `@/lib/auth/helpers`
   - Import `supabase` from `@/lib/db/client` (service_role for creating signed URLs)
   - Accept JSON body: `{ filename: string, contentType: string, size: number, chatId: string }`
   - Validate `size <= 10 * 1024 * 1024` (10MB, per D-04). Return 400 with `{ error: 'File too large. Maximum size is 10 MB.' }` if exceeded
   - Validate `contentType` starts with `image/` OR is `application/pdf` OR is `application/vnd.openxmlformats-officedocument.wordprocessingml.document`. Return 400 with `{ error: 'Unsupported file type' }` if not
   - Construct path: `${userId}/${chatId}/${Date.now()}-${filename}`
   - Call `supabase.storage.from('attachments').createSignedUploadUrl(path)`
   - Return JSON: `{ signedUrl: data.signedUrl, token: data.token, path }`
   - Handle errors: AuthenticationError -> 401, other -> 500

4. Create `app/api/upload/complete/route.ts` -- POST endpoint called after the client uploads the file to Storage. Handles document text extraction and DB record creation:
   - Import `requireAuth`, `supabase` from client, `createDocument` from `@/lib/db/documents`
   - Accept JSON body: `{ path: string, chatId: string, fileName: string, fileType: string }`
   - Construct public URL: `${process.env.NEXT_PUBLIC_SUPABASE_URL}/storage/v1/object/public/attachments/${path}`
   - If `fileType` is `application/pdf`:
     - Download the file from Storage using `supabase.storage.from('attachments').download(path)`
     - Extract text using pdf-parse v2: `import { PDFParse } from 'pdf-parse'`, create `new PDFParse({ data: new Uint8Array(buffer) })`, call `parser.getText({ pageJoiner: '\n\n' })`, call `parser.destroy()`
     - Truncate extracted text to 4000 chars if longer (per D-06): `text.slice(0, 4000) + (text.length > 4000 ? '... [truncated]' : '')`
     - Call `createDocument(chatId, fileName, fileType, publicUrl, extractedText)`
   - If `fileType` is `application/vnd.openxmlformats-officedocument.wordprocessingml.document`:
     - Download from Storage, extract text using `mammoth.extractRawText({ buffer })`
     - Truncate to 4000 chars same as PDF
     - Call `createDocument(chatId, fileName, fileType, publicUrl, extractedText)`
   - If `fileType` starts with `image/`:
     - No extraction needed. Return `{ fileUrl: publicUrl, type: 'image' }`
   - Return JSON: `{ fileUrl: publicUrl, type: 'image' | 'document', extractedText?: string }`
   - Handle errors: extraction failures should NOT block the upload -- save document without extracted text and return a warning

5. Update `next.config.ts` to add `pdf-parse` to `serverExternalPackages` (per research Pitfall 2, Open Question 2):
   ```typescript

     serverExternalPackages: ['pdf-parse'],
   };
   ```
