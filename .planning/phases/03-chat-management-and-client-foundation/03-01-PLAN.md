Create `hooks/use-chats.ts` as a client module (`'use client'` at top) with three exported hooks.

**useChats()** (per CHAT-02):
```typescript
import { useQuery } from '@tanstack/react-query'
import type { Chat } from '@/lib/types'

export function useChats() {
  return useQuery<Chat[]>({
    queryKey: ['chats'],
    queryFn: async () => {
      const res = await fetch('/api/chats')
      if (!res.ok) throw new Error('Failed to fetch chats')
      const data = await res.json()
      return data.chats
    },
  })
}
```

**useCreateChat()** (per CHAT-01):
Uses `useMutation` with optimistic insert pattern:
1. Import `useRouter` from `next/navigation` for post-create navigation
2. `mutationFn`: POST to `/api/chats` with `body: JSON.stringify({})` (empty body, no title -- CHAT-04 deferred to Phase 4). Headers: `{ 'Content-Type': 'application/json' }`. Parse response, return `data.chat as Chat`.
3. `onMutate`: Cancel queries for `['chats']`, snapshot previous chats, create temp chat with `crypto.randomUUID()` as id, empty string for `user_id`, `null` for title, `new Date().toISOString()` for `created_at` and `updated_at`. Prepend to existing chat list in cache. Return `{ previousChats }`.
4. `onSuccess(newChat)`: Navigate to `/chat/${newChat.id}` using `router.push()`. This navigates to the REAL server-assigned ID, not the temp one.
5. `onError(_err, _vars, context)`: Rollback by restoring `context.previousChats` to `['chats']` query data. Log `console.error('Failed to create chat. Please try again.')`.
6. `onSettled`: Invalidate `['chats']` query to sync with server.

**useDeleteChat()** (per CHAT-03):
Uses `useMutation` with optimistic removal pattern:
1. Import `useRouter` and `usePathname` from `next/navigation`
2. `mutationFn(chatId: string)`: DELETE to `/api/chats/${chatId}`. Throw on !res.ok.
3. `onMutate(chatId)`: Cancel queries for `['chats']`, snapshot previous chats, filter out the chat by id from cache. Return `{ previousChats }`.
4. `onSuccess(_data, chatId)`: If user was viewing the deleted chat (pathname includes chatId), navigate to `/` using `router.push('/')`.
5. `onError(_err, _chatId, context)`: Rollback by restoring `context.previousChats`. Log `console.error('Failed to delete chat. Please try again.')`.
6. `onSettled`: Invalidate `['chats']` query.

Full imports at top of file:
```typescript
'use client'

import { useMutation, useQuery, useQueryClient } from '@tanstack/react-query'
import { useRouter, usePathname } from 'next/navigation'
import type { Chat } from '@/lib/types'
```

Key patterns:
- Query key for list: `['chats']` (per CONTEXT.md locked decision)

- Error handling uses `console.error` (per UI-SPEC: toast formalized in Phase 4)
- `useDeleteChat` checks `usePathname()` to determine if the deleted chat is currently active, and navigates to `/` if so
