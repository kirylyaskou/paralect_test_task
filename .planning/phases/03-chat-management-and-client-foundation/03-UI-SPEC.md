# Phase 3 — UI Design Contract

> Visual and interaction contract for the chat sidebar, responsive layout shell, chat CRUD flows, and theme toggle. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn v4 |
| Preset | base-nova / neutral base color |
| Component library | Base UI (`@base-ui/react`) |
| Icon library | Lucide |
| Font | Inter (Google Fonts, loaded in root layout) |

Continuity note: All design tokens and conventions from Phase 2 carry forward unchanged. This phase extends the component inventory and introduces the application shell layout.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label in sidebar items |
| sm | 8px | Sidebar item internal padding, gap between sidebar header elements |
| md | 16px | Default content padding, sidebar group vertical gap |
| lg | 24px | Main content area padding, card inner padding |
| xl | 32px | Main content area horizontal padding on desktop |
| 2xl | 48px | Empty state vertical centering offset, touch target minimum |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Sidebar width: 256px (16rem) on desktop -- set by shadcn sidebar component default `--sidebar-width: 16rem`
- Sidebar width mobile: 288px (18rem) -- set by shadcn sidebar component default `--sidebar-width-mobile: 18rem`
- Sidebar menu item height: 32px (h-8) -- shadcn sidebar default
- Touch target minimum: 48px for mobile sidebar trigger button (exceeds WCAG 2.5.5 recommended 44px, maps to existing `2xl` token)

## Typography

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 (leading-snug) | Chat item title text in sidebar, empty state body, dialog body |
| Label | 14px (text-sm) | 600 (semibold) | 1.4 | Sidebar group labels, "New Chat" button text |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 (leading-tight) | Empty state heading in main content area |
| Caption | 12px (text-xs) | 400 (normal) | 1.5 | Chat item timestamp, sidebar footer text, skeleton placeholder |

**Weight constraint:** This phase uses exactly 2 weights -- 400 (normal) for body content and captions, 600 (semibold) for labels and headings. No other weights are permitted. This continues the Phase 2 constraint.

## Color

All colors use the existing shadcn CSS custom properties defined in `app/globals.css`. Do NOT introduce new color variables. Sidebar-specific tokens (`--sidebar`, `--sidebar-foreground`, etc.) are already defined in globals.css for both light and dark modes.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` | Main content area background |
| Secondary (30%) | `--sidebar` / `--sidebar-foreground` | Sidebar surface, sidebar text |
| Accent (10%) | `--primary` / `--primary-foreground` | "New Chat" button only |
| Destructive | `--destructive` | Delete chat button in confirmation dialog, destructive action text |
| Muted | `--muted-foreground` | Chat timestamps, empty state body text, disabled states |
| Border | `--sidebar-border` | Sidebar right border on desktop, separator lines |
| Sidebar accent | `--sidebar-accent` / `--sidebar-accent-foreground` | Hovered sidebar item, active (selected) chat item |

Accent reserved for: "New Chat" button in the sidebar header. No other elements use `--primary` in this phase. The active chat highlight uses `--sidebar-accent` (not `--primary`).

## Focal Point

The sidebar acts as the persistent navigation anchor. The "New Chat" button in the sidebar header is the primary visual focal point, distinguished by accent color (`--primary`) against the neutral sidebar surface. The active chat item is the secondary focal point, highlighted with `--sidebar-accent` background. On mobile, the sidebar trigger button (hamburger icon) in the top-left of the main content header is the entry point to navigation.

### Sidebar Structure

```
+---------------------------+
| SidebarHeader             |
|  [=== New chat ===]       |  <- Primary CTA button, full width, accent color
|---------------------------|
| SidebarContent            |
|  SidebarGroup             |
|   SidebarGroupLabel       |
|    "Recent chats"         |
|   SidebarGroupContent     |
|    SidebarMenu            |
|     [icon] Chat title  .. |  <- Each chat as SidebarMenuItem
|     [icon] Chat title  .. |     ".." is dropdown trigger (MoreHorizontal)
|     [icon] Chat title  .. |
|     ...                   |
|---------------------------|
| SidebarFooter             |
|  [user email]  [Sign out] |
+---------------------------+
```

**Sidebar header:**
- "New chat" button: full-width, `variant="default"` (accent/primary), Lucide `Plus` icon on left
- Gap: 8px between icon and text

**Sidebar content:**
- Group label: "Recent chats" in `text-xs font-medium text-sidebar-foreground/70`
- Each chat item: `SidebarMenuButton` with Lucide `MessageSquare` icon + truncated title
- Active chat: `isActive={true}` prop on `SidebarMenuButton` (applies `--sidebar-accent` bg + `font-medium`)
- Chat with no title: display "New chat" in `text-muted-foreground italic`
- Each chat item has a `SidebarMenuAction` with Lucide `MoreHorizontal` icon (appears on hover on desktop, always visible on mobile), with `aria-label="Chat options"` for accessibility
- Clicking `MoreHorizontal` opens a `DropdownMenu` with a single "Delete chat" item

**Sidebar footer:**
- User email displayed as truncated text (`text-sm text-sidebar-foreground`)
- "Sign out" button: `variant="ghost"`, `size="sm"`, Lucide `LogOut` icon

### Sidebar Chat Item

| State | Visual Treatment |
|-------|-----------------|
| Default | `--sidebar-foreground` text, transparent bg |
| Hover | `--sidebar-accent` bg, `--sidebar-accent-foreground` text |

| Loading (skeleton) | `SidebarMenuSkeleton` with `showIcon={true}`, random width shimmer |

### "New Chat" Button

| State | Visual Treatment |
|-------|-----------------|
| Default | `--primary` bg, `--primary-foreground` text, full width |
| Hover | `--primary/80` bg (shadcn default hover) |
| Loading | Lucide `Loader2` with `animate-spin`, button disabled |
| Disabled | 50% opacity, `pointer-events-none` |

### Theme Toggle Button

| State | Visual Treatment |
|-------|-----------------|
| Light mode | Lucide `Sun` icon displayed, `variant="ghost"` |
| Dark mode | Lucide `Moon` icon displayed, `variant="ghost"` |
| Hover | `--accent` bg (ghost button hover) |

Implementation: Use `useTheme()` from `next-themes`. Toggle between `"light"` and `"dark"`. The `ThemeProvider` is already configured in `providers.tsx` with `attribute="class"` and `defaultTheme="system"`.

### Delete Confirmation Dialog

| State | Visual Treatment |
|-------|-----------------|
| Trigger | "Delete chat" item in dropdown menu, `text-destructive` color |
| Dialog open | Centered modal with overlay backdrop |
| Keep chat button | `variant="outline"`, positioned left |
| Delete chat button | `variant="destructive"`, positioned right |
| Deleting | Delete chat button shows `Loader2` with `animate-spin`, both buttons disabled |

### Sidebar Mobile Trigger

| State | Visual Treatment |
|-------|-----------------|
| Default | Lucide `PanelLeft` icon, `variant="ghost"`, `size="icon-sm"` |
| Hover | `--accent` bg |
| Sidebar open | Sheet slides in from left with dimmed backdrop overlay |

### Optimistic Updates (TanStack Query)

| Action | Optimistic Behavior |
|--------|---------------------|
| Create chat | New chat item appears instantly at top of sidebar list; navigates to `/chat/[id]` on server confirmation |
| Delete chat | Chat item disappears instantly from sidebar; if deleted chat was active, navigate to `/` |
| Rollback on error | Item reappears (delete) or disappears (create) with error toast |

Error toast: Use a simple `div` notification positioned at bottom-center for now. Toast component will be formalized in Phase 4 (UX-03). For this phase, use `console.error` for failed optimistic rollbacks and restore the previous state silently. If a more visible error is needed, add a temporary inline alert.

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Mobile (< 768px) | Sidebar hidden by default; opens as Sheet overlay from left via `SidebarTrigger`. Main content takes full width. Header shows sidebar trigger on left, theme toggle on right. |
| Desktop (>= 768px) | Sidebar visible as fixed panel on left (256px). Main content fills remaining width. Sidebar can be collapsed/expanded via trigger or `Ctrl+B` keyboard shortcut. |

The 768px breakpoint (`md:`) aligns with the shadcn sidebar component's built-in `useIsMobile()` hook which returns true for `< 768px`.

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Sidebar toggle | `SidebarTrigger` has `sr-only` text "Toggle Sidebar" built in |
| Mobile Sheet | Sheet has `sr-only` SheetTitle "Sidebar" and SheetDescription built into shadcn sidebar |
| Keyboard shortcut | `Ctrl+B` / `Cmd+B` toggles sidebar (built into `SidebarProvider`) |
| Chat item navigation | Each `SidebarMenuButton` is focusable, navigable via Tab/Arrow keys |
| Active chat | `data-active` attribute set on active chat for assistive technology |
| Delete dialog | Dialog traps focus, Escape closes, "Keep chat" returns focus to trigger |
| Delete confirmation | Dialog uses `role="alertdialog"` for destructive action |
| Theme toggle | Button has tooltip text describing the action |
| Chat options trigger | `MoreHorizontal` icon button has `aria-label="Chat options"` for screen readers |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1) -- shadcn neutral palette satisfies this for both light and dark modes |
| Reduced motion | `animate-spin` on loaders respects `prefers-reduced-motion` via Tailwind defaults |

## Dark Mode Behavior

The `ThemeProvider` is already configured in `providers.tsx` with:
- `attribute="class"` -- adds/removes `.dark` class on `<html>`
- `defaultTheme="system"` -- respects OS preference on first visit
- `enableSystem={true}` -- responds to OS theme changes

Theme toggle cycles: system -> light -> dark -> system. Implementation uses `setTheme()` from `useTheme()` hook.

All colors in `globals.css` have both `:root` (light) and `.dark` variants defined. The sidebar-specific tokens (`--sidebar`, `--sidebar-foreground`, `--sidebar-accent`, etc.) also have dark variants.

No additional CSS is needed for dark mode support. All shadcn components inherit theme-aware tokens automatically.
