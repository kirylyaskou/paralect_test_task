# Requirements: Chatbot (Paralect Product Academy)

**Defined:** 2026-03-27
**Core Value:** Strict architectural separation (DB -> API -> Client) with zero database calls from components

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Authentication

- [ ] **AUTH-01**: User can create account with email and password
- [ ] **AUTH-02**: User can log in and session persists across browser refresh
- [ ] **AUTH-03**: User can log out from any page
- [ ] **AUTH-04**: Unauthenticated users are redirected to login page via middleware

### Chat Management

- [ ] **CHAT-01**: User can create a new chat
- [ ] **CHAT-02**: User can see list of their chats in sidebar sorted by recency
- [ ] **CHAT-03**: User can delete a chat (with confirmation)
- [ ] **CHAT-04**: Chat title is auto-generated by LLM from first message
- [ ] **CHAT-05**: User can only access their own chats (ownership validation)

### Messaging

- [ ] **MSG-01**: User can send a text message and receive AI response
- [ ] **MSG-02**: AI response streams token-by-token via SSE
- [ ] **MSG-03**: Messages render markdown with syntax-highlighted code blocks
- [ ] **MSG-04**: Full conversation history loads when opening a chat

### Image Attachments

- [ ] **IMG-01**: User can paste image from clipboard into chat input
- [ ] **IMG-02**: User can upload image file via button
- [ ] **IMG-03**: Image preview shown before sending
- [ ] **IMG-04**: Images sent to OpenAI vision API for analysis

### Document Attachments

- [ ] **DOC-01**: User can upload PDF/DOCX documents to a chat
- [ ] **DOC-02**: Extracted text is injected as context in LLM system prompt
- [ ] **DOC-03**: Attached documents are listed/indicated in the chat

### Anonymous Access

- [ ] **ANON-01**: Unauthenticated user can ask up to 3 questions
- [ ] **ANON-02**: Question count tracked server-side via fingerprint
- [ ] **ANON-03**: Registration prompt shown when limit reached

### Multi-Tab Sync

- [ ] **SYNC-01**: Chat list updates in real-time across browser tabs via Supabase Realtime

### UX Polish

- [ ] **UX-01**: Loading skeletons for sidebar and message list
- [ ] **UX-02**: Welcome screen with suggested prompts on empty chat
- [ ] **UX-03**: Error toasts and retry buttons on API failures
- [ ] **UX-04**: Responsive layout with sidebar as Sheet on mobile
- [ ] **UX-05**: Dark/light theme toggle

### Architecture

- [x] **ARCH-01**: Strict 3-layer separation: DB (`lib/db/`) -> API (`app/api/`) -> Client (components/hooks)
- [x] **ARCH-02**: Zero database calls from components (including Server Components)
- [x] **ARCH-03**: Supabase service_role key used only in `lib/db/client.ts`
- [ ] **ARCH-04**: All API routes validate authentication and return proper HTTP status codes
- [ ] **ARCH-05**: Input validation with Zod schemas on all API endpoints

### Deployment

- [ ] **DEPL-01**: Application deployed to Vercel
- [ ] **DEPL-02**: README with setup instructions, architecture diagram, and deploy link
- [ ] **DEPL-03**: Loom demo video (2-5 minutes)

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Chat Management

- **CHAT-06**: User can rename chat inline in sidebar
- **CHAT-07**: Keyboard shortcuts (Enter = send, Shift+Enter = newline)

### Messaging

- **MSG-05**: User can stop generation mid-stream
- **MSG-06**: Copy code button on code blocks
- **MSG-07**: Message regeneration (re-ask last question)

### Multi-Provider

- **PROV-01**: Gemini adapter for alternative LLM
- **PROV-02**: User can select model in settings

### Security

- **SEC-01**: Rate limiting on all API endpoints
- **SEC-02**: CSRF protection

## Out of Scope

| Feature | Reason |
|---------|--------|
| OAuth/social login | Email/password sufficient for demo |
| Real-time chat between users | Single-user chatbot, not messaging platform |
| Mobile native app | Web-only for this assignment |
| Message editing | Not in assignment spec |
| Payment/subscription | Anonymous limit is hardcoded |
| Voice input/output | Unnecessary complexity for demo |
| Plugin system | Out of scope for test assignment |
| Search across chats | Nice-to-have but not evaluator priority |
